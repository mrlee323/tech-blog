## OTA 개인화 추천 시스템
현재 많은 도메인에서 개인의 활동을 통해 상품을 추천해주는 시스템이 갖춰줘있다. 개인의 취향이나 성향을 따로 작성을 받아서 추천하기도하고, 사용자의 검색이나 상품 진입을 통해 해당 카테고리, 비슷한 상품들을 추천해주고있다. 현재 우리 서비스에는 해당 기능이 전혀 제공하고 있지않으며, 어떠한 기능도 활동을 수집하지도 않고있다. 현재 일부 배너 클릭, 페이지 뷰의 관련 로그를 수집하고있지만 통계를 위한 수집일뿐 해당 데이터를 가지고 기능을 제공하고있지않다. 

###1. 프론트개발자로서 해당 기능을 빠르게 도입 할 방법은?
**서버도 없고, DB도 없고 아무런 인프라 없이 빠르게 도입할 방법은 무엇일까 고민했다. 사용자의 활동을 가장 앞단에서 관찰하고 수집할수있는건 프론트이다. 유저 판별까지는 어려워도 지금 진입한 유저의 활동에는 수집하고 즉각적으로 반응은 해줄수 있지않을까 생각했다. localStorage를 활용하여 메인페이지에서 유저의 활동이나 보고있는 데이터들을 수집하여 추천시스템을 진행하기로 결정했다. 

###2. 사용자의 활동을 어떻게 판별하여 추천해줄것인가?
가장 쉬운방법은 직전 검색한 지역 , 호텔을 근거로 숙소를 추천해주는것이다. 이렇게 도입하면 추천하는 로직은 간결하고 쉬울수있으나 유저에게 보여지는 숙소들이 다양하지않고 매번 동일한 데이터들이 노출될것 같았다. 유저의 활동을 점수로 산출하여 어떤 숙소을 보여줄지 스코어 시스템을 추가하기로했다. 배너 클릭부터 검색, 숙소 상세페이지 진입, 예약페이지 진입등 유저 활동에 차등점수를 주고 시간의 흐름에 따라 최근 활동에 반영 계수를 적용해 좀더 높은 점수를 쌓일수 있도록 계산로직을 만들었다. 가장 높은 점수를 쌓은 지역이나, 호텔 관련 리스트를 추출할수있도록했다. 

###3. 사용자는 어떤 숙소를 추천받기를 원하나?
데이터를 수집을 하지만 개인 유저에게 맞지않는 숙소를 추천하면 무용지물 이 될것이다. 그래서 수집된 데이터에대해 어떤 호텔을 보여줄지 고민했다. 현재 우리 서비스에는 숙소의 관련 필터링 기능이 많이 부족한 상황이다. 현재 가능한 기능내에서 어떻게 숙소를 추출해 낼수있을까 고민했다. 현재 제공하고있는 성급, 숙소유형을 활용하고, 정렬기능 추천, 평점 순등을 조합해서 숙소 리스트를 추출하기로했다. 가장 중요한 가격 필터링이 있으면 유용할듯한데 아직 해당 기능이 구현되지않아 유저 활동 기반의 호텔들의 대략적인 금액대를 산출해서 호텔리스트에 일부 데이터를 필터링하거나 정렬해서 보여주는 방법으로 우선 적용해보았다. 또한 현재 api 로 개발되어있는기능은 지역 호텔리스트, 지도 geo 포인트기반의 호텔리스트 를 사용할수있었다. 특정 최종산출된 데이터에 지역 아이디 기준으로 호텔 리스트를 제공하기도하고, 특정 호텔 기준으로 반경 5km내에 호텔을 추출하여 제공하는 방향으로 잡았다. 여러 지역을 보고있으면 지역별 숙소를 비교할수있도록 지역탭 기준으로 호텔리스트를 보여주며 사용자 기반으로 다양하게 보여지고자 했다. 

###4. 우리가 앞으로 해야할일 
해당 기능을 서비스에 적용후 개인화 추천리스트가 사용자에게 얼마나 보여지고 사용자가 반응하는지 로그를 추가했다. 해당 로그를 통계를 가지고 앞으로 방향성을 정해보려고한다. 추천하는 리스트 유형중 어떤 타입에 사용자가 반응하는지, 해당 반응이 실 결제까지 이루어졌는지를 보고 앞으로의 추천시스템 방향을 정하려고 한다. 또한 현재는 localStorage에서만 관리를 하다보니 해당 유저가 다른 디바이스로 접근한다면 사용자의 활동을 수집의 연결성이 단절된다. 웹, 모웹 디바이스별, 비로그인에서로그인 시 동일 유저이면 계속적으로 활동이 연결되어 호텔을 추천해준다면 사용자의 편의성이 좋아질거라 생각한다. 현재 까지는 서버 없이 진행했는데 이부분은 서버 도입이 필요할지, 프론트개발자인 내가 어떻게 기능을 개발해야할지 고민중이다.
