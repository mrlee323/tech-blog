# 2026년을 시작하며: 복잡도와 생산성의 사이에서

지난 한 해는 서비스를 운영하는 프론트엔드 엔지니어로서 기술적 외연을 넓히는 동시에, 내실을 다지는 치열한 시간이었다. 다국어 서비스 런칭부터 결제 시스템 고도화, 대규모 트래픽 대응까지 실무의 최전선에서 다양한 과제들을 수행했다. 특히 기존에 두 명이 운영하던 서비스를 혼자 전담하며 개발부터 어드민 관리까지 도맡았던 경험은 엔지니어로서 책임감과 시야를 확장하는 계기가 되었다.

## 1. 트래픽 과부하 대응: 호텔 상세 페이지 성능 최적화

우리 서비스는 '셀럽 공동구매'라는 특성상 특정 시간에 트래픽이 폭발적으로 몰리는 구조를 가지고 있습니다. 오픈 직후 상세 페이지 진입 시 발생하는 과도한 API 호출로 서버 부하가 발생하는 문제를 프론트엔드 관점에서 개선했습니다.

### 주요 작업
* **React Query 기반 캐싱 레이어 도입**: 동일 데이터에 대한 중복 호출을 차단하고, 효율적인 캐싱 전략을 통해 네트워크 비용을 절감했습니다.
* **데이터 로딩 전략 수정**: 모든 데이터를 한 번에 불러오지 않고, 사용자 시놉시스에 맞춘 **지연 로딩(Lazy Loading)**을 적용하여 초기 서버 부하를 분산했습니다.

### 성과 및 배운 점
* 서버 트래픽 부하를 유의미하게 줄여 서비스 안정성을 확보했습니다.
* 성능 최적화는 단순히 속도를 높이는 작업이 아니라, **전체 인프라 자원을 효율적으로 배분하는 엔지니어링의 정수**임을 깨달았습니다.

## 2. 글로벌 확장: 다국어 서비스 및 복합 통화 결제 시스템 구축

기존 서비스를 일본어와 인도네시아어로 확장하며, 국가별로 상이한 통화 및 결제 환경을 표준화하는 작업을 진행했습니다.

### 주요 도전 과제
* **복합 통화 및 정산 로직**: 국가별로 다른 할인(쿠폰, 프로모션) 및 적립금 적용 로직을 해당 통화에 맞춰 정밀하게 환산 처리했습니다.
* **데이터 정합성 확보**: 현지 통화 노출과 실제 달러($) 결제 사이의 오차를 방지하기 위해 소수점 처리 로직을 검증하고 서버와 정밀하게 협업했습니다.
* **해외 결제 수단 연동**: WeChat Pay, Alipay 등 신규 결제 수단을 연동하며 각 환경에 맞는 결제 플로우를 공통 인터페이스로 추상화했습니다.

## 3. 결제 시스템 리팩토링: 복잡한 로직의 객체화

결제 수단이 늘어남에 따라 파편화된 로직은 유지보수의 큰 걸림돌이었습니다. 이를 개선하기 위해 결제 프로세스 전반을 리팩토링했습니다.

### 개선 사항
* **결제 로직 객체화**: 각 결제 수단별 특수성을 추상화하여 하나의 공통 인터페이스(객체)로 관리할 수 있도록 구조화했습니다.
* **프로세스 단일화**: 결제 폼 생성부터 상태 처리(성공, 실패, 완료)에 이르는 흐름을 정형화하여 코드 가독성과 확장성을 높였습니다.

## 4. AI와 생산성: 의존이 아닌 '확장'의 도구로

가파른 업무량을 소화할 수 있었던 원동력 중 하나는 AI 도구의 적극적인 활용이었습니다. 작년 5월 Cursor를 도입한 이후, 단순 코드 작성을 넘어 설계의 파트너로서 AI를 실험해왔습니다. 

특히 서비스 리뉴얼 후 3년이 지나며 비대해진 레거시와 복잡한 사이드 이팩트를 파악하는 데 AI를 활용했습니다. 내가 설계한 로직의 빈틈을 체크하고 예기치 못한 영향도를 검증받으며, 복잡도가 높은 코드를 개선하는 데 실질적인 도움을 받았습니다.

## 2026년, 엔지니어로서의 지향점

최근에는 단순히 AI를 사용하는 것을 넘어, 어떻게 하면 반복적인 업무 패턴을 자동화하고 나의 리소스를 더 고부가가치 작업에 투입할 수 있을지 고민하고 있습니다. 

AI가 모든 것을 커버할 수 없는 영역, 즉 **'비즈니스의 복잡도를 기술적으로 어떻게 추상화할 것인가'**에 더 집중하려 합니다. 2026년은 도구의 발전에 휘둘리기보다, 그 도구를 활용해 더 견고하고 지속 가능한 아키텍처를 설계하는 안목을 기르는 데 집중하는 한 해가 될 것입니다.
